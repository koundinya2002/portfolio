Student Dashbord

The goal was to create an in-house application for communication between lecturers and students. Functionality of the application is simple, the users are categorized into 2 groups, teacher & student. Since this is for college, the users are created by the admin and the credentials are shared with the users. The users have an option to change only their password upon logging into their account. Initially the lecturer creates a classroom and shares the classroom code which is generated randomly. The students now can join the classroom using that code. The functionality of module is role based, lecturers have read and write permissions whereas students only have read permissions. The lecturer in a classroom can upload notes, publish a couple of bullet points with regards to the classroom & lecture, create assignments, update students with notifications. On the students can only view these and download the notes & notifications. Other modules include lost & found where users can ping an item found on campus for the people to find them. Another module include in-house social media exclusive for the college where users can post and share their media and thoughts, interact with other users via likes and comments.

Tech Stack: Django for backend; HTML5, CSS, JS for the front-end.
